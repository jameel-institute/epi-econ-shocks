This README explains files in the `data` directory.

There are two subdirectories, `data/raw/` and `data/outputs/`.
`data/raw/` holds data considered to be primary, including being provided by external parties.
`data/outputs/` holds data generated by processing the contents of `data/raw/`.
Currently, all data in `data/raw/` is ignored for by version control and hence for upload to GitHub.
Data in `data/outputs/` should be ignored for version control on a case by case basis.
If a file in `data/outputs/` is too similar in contents to one in `data/raw/`, do not track and upload it.
